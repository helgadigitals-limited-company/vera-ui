# Form

A collection of form components built on React Hook Form that provides validation, accessibility, and consistent styling. These components work together to create comprehensive, accessible forms with built-in error handling.

<Callout type="info">
  **Zod Integration**: All form examples in this documentation use Zod for schema validation, providing type-safe validation with excellent developer experience. The form components work seamlessly with React Hook Form and Zod resolver for robust form validation.
</Callout>

## Architecture

The form system consists of several components that work together:

- **`Form`** - Provides form context (alias for React Hook Form's `FormProvider`)
- **`FormField`** - Wraps React Hook Form's `Controller` with field context
- **`FormItem`** - Container for individual form fields with spacing
- **`FormLabel`** - Accessible label with error styling
- **`FormControl`** - Wrapper for form inputs with ARIA attributes
- **`FormDescription`** - Helper text for form fields
- **`FormMessage`** - Error and validation messages

### useFormField Hook

Custom hook that provides field state and accessibility IDs.

**Returns:**
```tsx
{
  id: string                    // Unique field ID
  name: string                  // Field name
  formItemId: string           // ID for form item
  formDescriptionId: string    // ID for description
  formMessageId: string        // ID for error message
  invalid: boolean             // Whether field has errors
  isDirty: boolean            // Whether field value changed
  isTouched: boolean          // Whether field was focused
  error?: FieldError          // Field error object
}
```

## Features

### Accessibility
- **ARIA Attributes**: Proper `aria-describedby`, `aria-invalid`, and `id` associations
- **Error Announcements**: Screen readers announce validation errors
- **Label Association**: Labels properly associated with form controls
- **Focus Management**: Logical tab order and focus indicators

### Validation
- **Built-in Rules**: Required, pattern, min/max length, custom validation
- **Async Validation**: Support for server-side validation
- **Real-time Feedback**: Validation on blur, change, or submit
- **Error Display**: Consistent error message styling and positioning

### Developer Experience
- **TypeScript Support**: Full type safety with form schemas
- **Schema Validation**: Works with Zod, Yup, Joi, and other validation libraries
- **Flexible Layout**: Compose forms with any layout structure
- **Performance**: Optimized re-renders and validation timing

## Accessibility

The form components provide comprehensive accessibility features:

- **Screen Reader Support**: Proper ARIA labels, descriptions, and error announcements
- **Keyboard Navigation**: Full keyboard support for all form elements
- **Focus Management**: Logical tab order and visible focus indicators
- **Error Handling**: Clear error messages with proper associations
- **Semantic HTML**: Uses appropriate form elements and structure

<Callout type="info">
  **Best Practice**: Always use `FormLabel`, `FormDescription`, and `FormMessage` together with `FormControl` to ensure proper accessibility and consistent styling.
</Callout>
