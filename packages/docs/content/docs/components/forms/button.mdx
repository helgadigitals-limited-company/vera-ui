# Button

A fundamental interactive element that triggers an action when activated. The Button component supports multiple variants, sizes, and states to fit different use cases in your application.

## Installation

```bash npm2yarn
npm install @helgadigitals/vera-ui
```

## Usage

```tsx
import { Button } from "@helgadigitals/vera-ui"

export default function Example() {
  return <Button>Click me</Button>
}
```

## API Reference

### Button

The main button component that renders an interactive element.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "destructive" \| "outline" \| "secondary" \| "ghost" \| "link"` | `"default"` | The visual style variant of the button |
| `size` | `"default" \| "sm" \| "lg" \| "icon"` | `"default"` | The size of the button |
| `asChild` | `boolean` | `false` | Change the default rendered element for the one passed as a child |
| `disabled` | `boolean` | `false` | When true, prevents the user from interacting with the button |

### Variant Details

- **default**: The primary button style with solid background
- **destructive**: For dangerous actions like delete operations
- **outline**: Button with border and transparent background  
- **secondary**: Alternative solid style with muted appearance
- **ghost**: Minimal style with no background or border
- **link**: Appears as a text link

### Size Details

- **sm**: Small button (32px height)
- **default**: Standard button (40px height)  
- **lg**: Large button (44px height)
- **icon**: Square button for icons (40px √ó 40px)

<Callout type="warn">
When using the `asChild` prop, make sure the child element can receive click events and accessibility properties.
</Callout>

## Accessibility

The Button component includes built-in accessibility features:

- Proper ARIA attributes
- Keyboard navigation support (Enter and Space keys)
- Focus management
- Screen reader compatibility

When disabled, the button automatically:
- Sets `aria-disabled="true"`
- Prevents focus and interaction
- Uses appropriate disabled styling

### Confirmation Button

```tsx
import { Button } from "@helgadigitals/vera-ui"
import { Trash2 } from "lucide-react"

export default function ConfirmationExample() {
  const handleDelete = () => {
    if (confirm("Are you sure you want to delete this item?")) {
      // Delete logic here
    }
  }

  return (
    <Button variant="destructive" onClick={handleDelete}>
      <Trash2 className="mr-2 h-4 w-4" />
      Delete Item
    </Button>
  )
}
```

## API Reference

### Props

The Button component accepts all standard HTML button attributes plus these additional props:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default"` \| `"destructive"` \| `"outline"` \| `"secondary"` \| `"ghost"` \| `"link"` | `"default"` | The visual style variant of the button |
| `size` | `"default"` \| `"sm"` \| `"lg"` \| `"icon"` | `"default"` | The size of the button |
| `asChild` | `boolean` | `false` | When true, the button will render as its child element |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `className` | `string` | - | Additional CSS classes |

### Styling

The Button component uses CSS variables for theming. You can customize the appearance by overriding these variables:

```css
.my-custom-button {
  --button-height: 2.5rem;
  --button-padding-x: 1rem;
  --button-font-size: 0.875rem;
}
```

## Accessibility

The Button component follows WAI-ARIA guidelines:

- **Keyboard Navigation**: Fully accessible via keyboard (Enter/Space to activate)
- **Screen Readers**: Proper semantic button element with accessible text
- **Focus Management**: Clear focus indicators and focus trapping when needed
- **Disabled State**: Properly communicated to assistive technologies

<Callout type="info">
  **ARIA Labels**: For icon-only buttons, always provide an `aria-label` or `aria-describedby` attribute for screen readers.
</Callout>

```tsx
// Good: Icon button with accessible label
<Button size="icon" aria-label="Add new item">
  <PlusIcon className="h-4 w-4" />
</Button>

// Better: Icon button with tooltip
<Tooltip>
  <TooltipTrigger asChild>
    <Button size="icon">
      <PlusIcon className="h-4 w-4" />
    </Button>
  </TooltipTrigger>
  <TooltipContent>Add new item</TooltipContent>
</Tooltip>
```
