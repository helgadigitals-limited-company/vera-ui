# Sidebar

A comprehensive navigation component that provides a collapsible sidebar with support for icons, badges, grouped items, and responsive behavior. Perfect for application dashboards and admin panels.

## Installation

```bash npm2yarn
npm install @helgadigitals/vera-ui lucide-react
```

## Usage

The Sidebar component requires a SidebarProvider wrapper to manage state:

```tsx
import { 
  SidebarProvider, 
  SidebarInset,
  SidebarTrigger 
} from "@helgadigitals/vera-ui"

export default function SidebarExample() {
  return (
    <SidebarProvider>
      <SidebarInset>
        <header className="flex h-16 shrink-0 items-center gap-2 border-b px-4">
          <SidebarTrigger className="-ml-1" />
          <h1>My Application</h1>
        </header>
        <div className="flex flex-1 flex-col gap-4 p-4">
          <div className="grid auto-rows-min gap-4 md:grid-cols-3">
            <div className="aspect-video rounded-xl bg-muted/50" />
            <div className="aspect-video rounded-xl bg-muted/50" />
            <div className="aspect-video rounded-xl bg-muted/50" />
          </div>
          <div className="min-h-[100vh] flex-1 rounded-xl bg-muted/50 md:min-h-min" />
        </div>
      </SidebarInset>
    </SidebarProvider>
  )
}
```

## API Reference

### SidebarProvider

The root provider component that manages sidebar state and context.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultOpen` | `boolean` | `true` | Default open state |
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when open state changes |

### Sidebar

The main sidebar container component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"left" \| "right"` | `"left"` | Which side of the screen to show the sidebar |
| `variant` | `"sidebar" \| "floating" \| "inset"` | `"sidebar"` | The sidebar variant |
| `collapsible` | `"offcanvas" \| "icon" \| "none"` | `"offcanvas"` | How the sidebar should behave when collapsed |

### SidebarContent

Container for sidebar content sections.

### SidebarHeader / SidebarFooter

Optional header and footer sections for the sidebar.

### SidebarGroup

Groups related sidebar items together.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### SidebarGroupLabel

Label for a sidebar group section.

### SidebarGroupContent

Container for group content.

### SidebarMenu

Container for sidebar menu items.

### SidebarMenuItem

Individual menu item wrapper.

### SidebarMenuButton

Interactive button for menu items.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as child element |
| `isActive` | `boolean` | `false` | Whether the item is currently active |
| `tooltip` | `string \| ComponentProps<typeof TooltipContent>` | - | Tooltip content when collapsed |

### SidebarTrigger

Button to toggle sidebar open/closed state.

### SidebarInset

Main content area that adjusts for sidebar presence.

### useSidebar

Hook to access sidebar state and controls.

#### Returns

| Property | Type | Description |
|----------|------|-------------|
| `state` | `"expanded" \| "collapsed"` | Current sidebar state |
| `open` | `boolean` | Whether sidebar is open |
| `setOpen` | `(open: boolean) => void` | Function to set open state |
| `openMobile` | `boolean` | Whether mobile sidebar is open |
| `setOpenMobile` | `(open: boolean) => void` | Function to set mobile open state |
| `isMobile` | `boolean` | Whether currently in mobile view |
| `toggleSidebar` | `() => void` | Function to toggle sidebar |

<Callout type="info">
The sidebar automatically handles mobile responsiveness and provides a consistent experience across all device sizes.
</Callout>

### Responsive Collapsible Modes

The sidebar supports different collapsible behaviors:

```tsx
// Icon mode - collapses to icons on desktop
<Sidebar collapsible="icon">

// Offcanvas mode - hides completely when collapsed  
<Sidebar collapsible="offcanvas">

// None - never collapses
<Sidebar collapsible="none">
```

## API Reference

### SidebarProvider

The root provider that manages sidebar state.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultOpen` | `boolean` | `true` | Initial open state |
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | Open state change handler |

### Sidebar

The main sidebar container.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"left"` \| `"right"` | `"left"` | Which side to display the sidebar |
| `variant` | `"sidebar"` \| `"floating"` \| `"inset"` | `"sidebar"` | Visual variant |
| `collapsible` | `"offcanvas"` \| `"icon"` \| `"none"` | `"offcanvas"` | Collapse behavior |

### SidebarMenuButton

Interactive menu button component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as child element |
| `isActive` | `boolean` | `false` | Active state |
| `variant` | `"default"` \| `"outline"` | `"default"` | Visual variant |
| `size` | `"default"` \| `"sm"` \| `"lg"` | `"default"` | Button size |
| `tooltip` | `string` \| `object` | - | Tooltip content |

## Accessibility

The Sidebar component follows WAI-ARIA guidelines:

- **Keyboard Navigation**: Full keyboard support with arrow keys and tab navigation
- **Screen Readers**: Proper semantic navigation structure with landmarks
- **Focus Management**: Focus trapping and restoration when collapsing/expanding
- **ARIA Labels**: Proper labeling for navigation regions and interactive elements

<Callout type="info">
  **Navigation Landmarks**: The sidebar automatically uses the `navigation` landmark role for better screen reader navigation.
</Callout>
