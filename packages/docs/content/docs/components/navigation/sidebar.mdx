# Sidebar & SidebarLayout

The Vera UI library provides a powerful and flexible sidebar system with two main components: `ReusableSidebar` and `SidebarLayout`. This comprehensive guide covers all functionalities and usage patterns, including the new **Mixed Content Support**.

<Callout type="info">
The sidebar system has been completely redesigned to provide maximum flexibility with support for mixed content types, auto-detection, and advanced styling options.
</Callout>

## Installation

```bash npm2yarn
npm install @helgadigitals/vera-ui lucide-react
```

## Usage

```tsx
import {
  SidebarLayout,
  ReusableSidebar,
  type SidebarItem,
  type Group,
  type MixedSidebarItem
} from "@helgadigitals/vera-ui";
import { Home, Calendar, Users, Settings } from "lucide-react";
```

## Data Types

### SidebarItem

```tsx
type SidebarItem = {
  title: string;           // Display name
  path: string;           // Route path
  icon: React.ElementType; // Lucide icon component
  exact?: boolean;        // Exact route matching
  badge?: string | number; // Optional badge/count
  tooltip?: string;       // Optional tooltip text
};
```

### Group

```tsx
type Group = {
  key: string;             // Unique identifier
  label: string;           // Group display name
  icon?: React.ElementType; // Optional icon
  items: SidebarItem[];    // Array of sidebar items
};
```

### MixedSidebarItem

```tsx
type MixedSidebarItem = SidebarItem | Group;
```

This union type allows you to mix individual sidebar items and groups in the same array for maximum flexibility.

## Basic Usage

### 1. Simple Sidebar with Direct Items

When you pass `SidebarItem[]`, the sidebar renders items directly without grouping:

```tsx
import { SidebarLayout } from "@helgadigitals/vera-ui";
import { Home, Calendar, Users, Settings } from "lucide-react";

const items: SidebarItem[] = [
  { title: "Home", path: "/", icon: Home },
  { title: "Calendar", path: "/calendar", icon: Calendar, badge: 3 },
  { title: "Users", path: "/users", icon: Users },
  { title: "Settings", path: "/settings", icon: Settings }
];

function App() {
  return (
    <SidebarLayout props={{ items }}>
      <div>Your main content here</div>
    </SidebarLayout>
  );
}
```

### 2. Grouped Sidebar

When you pass `Group[]`, the sidebar renders with collapsible groups:

```tsx
const groupedItems: Group[] = [
  {
    key: "main",
    label: "Main Navigation",
    items: [
      { title: "Dashboard", path: "/dashboard", icon: Home },
      { title: "Analytics", path: "/analytics", icon: BarChart }
    ]
  },
  {
    key: "admin",
    label: "Administration", 
    items: [
      { title: "Users", path: "/users", icon: Users, badge: 12 },
      { title: "Settings", path: "/settings", icon: Settings }
    ]
  }
];

function App() {
  return (
    <SidebarLayout props={{ items: groupedItems }}>
      <div>Your main content here</div>
    </SidebarLayout>
  );
}
```

## Mixed Content Support

The most powerful feature allows you to **mix individual `SidebarItem`s and `Group`s in the same array**:

### Basic Mixed Usage

```tsx
import { SidebarLayout, type MixedSidebarItem } from "@helgadigitals/vera-ui";
import { Home, Users, Settings, Profile, BarChart, FileText, Zap } from "lucide-react";

const mixedItems: MixedSidebarItem[] = [
  // Direct items appear at the top level
  { title: "Home", path: "/", icon: Home },
  { title: "Profile", path: "/profile", icon: Profile },
  
  // Groups appear below direct items
  {
    key: "admin",
    label: "Administration",
    items: [
      { title: "Users", path: "/users", icon: Users },
      { title: "Settings", path: "/settings", icon: Settings }
    ]
  },
  
  // Another direct item
  { title: "Quick Action", path: "/quick", icon: Zap },
  
  // Another group
  {
    key: "reports",
    label: "Reports",
    icon: BarChart,
    items: [
      { title: "Analytics", path: "/analytics", icon: BarChart },
      { title: "Documents", path: "/documents", icon: FileText }
    ]
  }
];

function App() {
  return (
    <SidebarLayout props={{ items: mixedItems }}>
      <div>Your main content here</div>
    </SidebarLayout>
  );
}
```

## Auto-Detection Logic

The sidebar automatically detects the data structure and renders accordingly:

- **Pure `SidebarItem[]`**: Renders flat list of navigation items
- **Pure `Group[]`**: Renders with collapsible group sections  
- **Mixed `MixedSidebarItem[]`**: Renders direct items first, then groups

<Callout type="info">
The auto-detection uses utility functions from `@helgadigitals/vera-ui` to determine the correct rendering mode.
</Callout>

## Props Reference

### SidebarLayout Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `props.items` | `SidebarItem[] \| Group[] \| MixedSidebarItem[]` | - | Navigation items configuration |
| `children` | `ReactNode` | - | Main content area |

### SidebarItem Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | Display name for the sidebar item |
| `path` | `string` | - | Route path for navigation |
| `icon` | `React.ElementType` | - | Lucide icon component |
| `exact` | `boolean` | `false` | Enable exact route matching |
| `badge` | `string \| number` | - | Optional badge/count display |
| `tooltip` | `string` | - | Optional tooltip text |

### Group Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `key` | `string` | - | Unique identifier for the group |
| `label` | `string` | - | Display name for the group |
| `icon` | `React.ElementType` | - | Optional icon for the group header |
| `items` | `SidebarItem[]` | - | Array of sidebar items in the group |

## API Reference

### SidebarProvider

The root provider component that manages sidebar state and context.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultOpen` | `boolean` | `true` | Default open state |
| `open` | `boolean` | - | Controlled open state |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when open state changes |

### Sidebar

The main sidebar container component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"left" \| "right"` | `"left"` | Which side of the screen to show the sidebar |
| `variant` | `"sidebar" \| "floating" \| "inset"` | `"sidebar"` | The sidebar variant |
| `collapsible` | `"offcanvas" \| "icon" \| "none"` | `"offcanvas"` | How the sidebar should behave when collapsed |

### SidebarContent

Container for sidebar content sections.

### SidebarHeader / SidebarFooter

Optional header and footer sections for the sidebar.

### SidebarGroup

Groups related sidebar items together.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### SidebarGroupLabel

Label for a sidebar group section.

### SidebarGroupContent

Container for group content.

### SidebarMenu

Container for sidebar menu items.

### SidebarMenuItem

Individual menu item wrapper.

### SidebarMenuButton

Interactive button for menu items.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as child element |
| `isActive` | `boolean` | `false` | Whether the item is currently active |
| `tooltip` | `string \| ComponentProps<typeof TooltipContent>` | - | Tooltip content when collapsed |

### SidebarTrigger

Button to toggle sidebar open/closed state.

### SidebarInset

Main content area that adjusts for sidebar presence.

### useSidebar

Hook to access sidebar state and controls.

#### Returns

| Property | Type | Description |
|----------|------|-------------|
| `state` | `"expanded" \| "collapsed"` | Current sidebar state |
| `open` | `boolean` | Whether sidebar is open |
| `setOpen` | `(open: boolean) => void` | Function to set open state |
| `openMobile` | `boolean` | Whether mobile sidebar is open |
| `setOpenMobile` | `(open: boolean) => void` | Function to set mobile open state |
| `isMobile` | `boolean` | Whether currently in mobile view |
| `toggleSidebar` | `() => void` | Function to toggle sidebar |

<Callout type="info">
The sidebar automatically handles mobile responsiveness and provides a consistent experience across all device sizes.
</Callout>

### Responsive Collapsible Modes

The sidebar supports different collapsible behaviors:

```tsx
// Icon mode - collapses to icons on desktop
<Sidebar collapsible="icon">

// Offcanvas mode - hides completely when collapsed  
<Sidebar collapsible="offcanvas">

// None - never collapses
<Sidebar collapsible="none">
```

## Accessibility

The Sidebar component follows WAI-ARIA guidelines:

- **Keyboard Navigation**: Full keyboard support with arrow keys and tab navigation
- **Screen Readers**: Proper semantic navigation structure with landmarks
- **Focus Management**: Focus trapping and restoration when collapsing/expanding
- **ARIA Labels**: Proper labeling for navigation regions and interactive elements

<Callout type="info">
  **Navigation Landmarks**: The sidebar automatically uses the `navigation` landmark role for better screen reader navigation.
</Callout>
