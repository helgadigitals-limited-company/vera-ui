### Code Quality

- **Error handling** - Handle missing routes and invalid paths gracefully

## Advanced Patterns

### Dynamic Navigation

Generate navigation items based on user permissions:

```tsx
import { ReusableSidebar, SidebarProvider } from '@helgadigitals/vera-ui';
import { Home, Users, Settings } from 'lucide-react';

function DynamicSidebar({ userRole }: { userRole: string }) {
  const items = React.useMemo(() => {
    const base = [
      { title: "Dashboard", path: "/", icon: Home }
    ];
    
    if (userRole === 'admin') {
      base.push(
        { title: "Users", path: "/users", icon: Users },
        { title: "Settings", path: "/settings", icon: Settings }
      );
    }
    
    return base;
  }, [userRole]);

  return (
    <SidebarProvider>
      <ReusableSidebar items={items} heading="My App" />
    </SidebarProvider>
  );
}
```

```tsx
import { ReusableSidebar, SidebarProvider } from '@helgadigitals/vera-ui';

```

## Troubleshooting

### Sidebar not showing active state

**Problem:** Navigation items don&apos;t highlight when navigating to their routes.

**Solution:** Ensure you&apos;re using React Router or that `window.location.pathname` is updated correctly. The component relies on the pathname to determine active state.

```tsx
// Make sure your router is configured correctly
import { BrowserRouter } from 'react-router-dom';

<BrowserRouter>
  <SidebarProvider>
    <ReusableSidebar items={items} />
  </SidebarProvider>
</BrowserRouter>
```

### Groups not persisting state

**Problem:** Group open/closed states reset on page refresh.

**Solution:** The component automatically saves state to localStorage. If it&apos;s not working, check that:
1. localStorage is available (not in incognito/private mode)
2. Each group has a unique `key` property
3. Browser isn&apos;t blocking localStorage access

### Icons not displaying

**Problem:** Navigation icons appear as empty spaces.

**Solution:** Install and import lucide-react icons correctly:

```bash
pnpm add lucide-react
```

```tsx
import { Home, Users, Settings } from 'lucide-react';
// Not: import * as Icons from 'lucide-react'
```

### Sidebar overflow issues

**Problem:** Sidebar content is cut off or not scrollable.

**Solution:** Ensure parent container has proper height constraints:

```tsx
<div className="flex h-screen"> {/* or h-full with parent height set */}
  <ReusableSidebar items={items} />
  <main className="flex-1 overflow-auto">
    {content}
  </main>
</div>
```
